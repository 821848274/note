| :--------------- | :---------- | :----------------------- | :----------------------------------------------------------- |
| **逻辑取及输出** | **LD**      | **Load**                 | **取指令**。用于常开触点与左母线的连接，表示一个逻辑运算的开始 。 |
|                  | **LDI**     | **Load Inverse**         | **取反指令**。用于常闭触点与左母线的连接，表示一个逻辑运算的开始。 |
|                  | **OUT**     | **Out**                  | **输出指令**。将逻辑运算的结果驱动到指定的线圈，如输出继电器Y、辅助继电器M等。 |
| **触点串联**     | **AND**     | **And**                  | **与指令**。串联一个常开触点，实现逻辑"与"运算。             |
|                  | **ANI**     | **And Inverse**          | **与非指令**。串联一个常闭触点，实现逻辑"与非"运算。         |

|                  | **ANB**     | **And Block**            | **块与指令**。用于将一个并联电路块（由多个触点并联而成）与前面的电路串联。 |
| **置位与复位**   | **SET**     | **Set**                  | **置位指令**。使目标元件（如Y、M）接通（置1）并保持该状态。  |
|                  | **RST**     | **Reset**                | **复位指令**。使目标元件（如Y、M、T、C）断开（置0）或清除数据寄存器的内容。 |

|                  | **PLF**     | **Pulse Falling**        | **下降沿脉冲指令**。在输入信号由ON→OFF的下降沿时，产生一个扫描周期的脉冲输出。 |
| **主控指令**     | **MC**      | **Master Control**       | **主控指令**。用于表示公共逻辑条件（主控触点）的开始，可以简化多个线圈共用同一触点的电路。 |
|                  | **MCR**     | **Master Control Reset** | **主控复位指令**。用于表示主控区的结束，使母线返回到原来的位置。 |
| **栈操作**       | **MPS**     | **Push (Point storage)** | **进栈指令**。将当前的运算结果压入堆栈存储器的第一段（栈顶）。 |
|                  | **MRD**     | **Read**                 | **读栈指令**。读出堆栈存储器最上级（栈顶）的数据，堆栈内数据不发生移动。 |
|                  | **MPP**     | **Pop**                  | **出栈指令**。读出并弹出堆栈存储器最上级（栈顶）的数据，堆栈内其他数据顺序上移。 |
| **程序控制**     | **NOP**     | **No Operation**         | **空操作指令**。无任何具体动作，通常用于预留程序步或清除指令。 |
|                  | **END**     | **End**                  | **结束指令**。表示程序到此结束，PLC执行到END后立即返回进行输出处理。 |

### 步进指令

![image-20260327183931080](.\assets\image-20260327183931080.png)



#### 1. 什么是步进指令？

步进指令包括两条：

- **STL（Step Ladder）**：步开始指令，表示一个“状态步”的起点。
- **RET（Return）**：步结束指令，表示步进程序结束，返回到母线。

它们必须与**状态继电器（S）**配合使用，将一个复杂的流程分解成一个个独立、顺序执行的“步骤”。

> 通俗理解：就像一张流程图，每个“框”是一个步，框内有该步要做的动作，框与框之间用转移条件连接。当前步执行完后，满足条件就进入下一步，同时自动清除前一步的动作。

------

#### 2. 状态继电器（S）

三菱FX3U中状态继电器编号范围：**S0~S4095**，常用分类：

| 类型     | 编号范围  | 特点                                              |
| :------- | :-------- | :------------------------------------------------ |
| 初始步   | S0~S9     | 用于程序起始步，必须用**IST**或**初始化脉冲**激活 |
| 回原点步 | S10~S19   | 用于机械回零流程                                  |
| 通用步   | S20~S499  | 普通步，断电不保持                                |
| 保持步   | S500~S899 | 断电保持（停电记忆）                              |
| 报警步   | S900~S999 | 用于故障报警                                      |

每个步只能**激活一次**（当前步），转移后原步自动复位（其内的输出也自动复位）。

------

#### 3. STL 指令的用法

**语法**：

plaintext

```
STL  S20     ; 步S20开始
...          ; 该步的控制程序
OUT  Y0      ; 该步的输出
LD   X0      ; 转移条件
SET  S21     ; 转移到下一步S21
```



**关键点**：

- STL S__ 相当于一个**常开触点**，只有当S__被**置位（SET）**时，该触点闭合，其后的梯形图才执行。
- 在一个STL步内，可以编写：
  - 输出线圈（Y、M）
  - 定时器（T）、计数器（C）
  - 条件跳转（用SET Snext 转移到下一步）
  - 分支选择（根据不同条件转移到不同步）
- 当用`SET Snext`转移到下一步后，当前步会自动**复位**（S__=OFF），且该步内的**非保持型输出**（如OUT Y0）自动变为OFF。
  - 但`SET` / `RST`指令设置的元件会保持状态。

------

#### 4. RET 指令

- 必须放在所有STL步的**最后面**，表示步进程序结束。
- 之后可以写普通梯形图（非步进程序）。
- **如果漏掉RET，PLC会报错**。

------

#### 5. 步进程序的完整结构

一个典型的步进程序框架：

plaintext

```
       LD    M8002        ; 初始化脉冲（上电瞬间）
       SET   S0           ; 激活初始步S0

STL    S0                 ; 初始步（一般用来等待启动信号）
       LD    X0           ; 启动按钮
       SET   S20          ; 进入第1个动作步

STL    S20                ; 第1步
       OUT   Y0           ; 动作1
       OUT   T0   K50     ; 定时5秒
       LD    T0
       SET   S21          ; 时间到转下一步

STL    S21                ; 第2步
       OUT   Y1
       OUT   T1   K30
       LD    T1
       SET   S22

STL    S22                ; 最后一步
       OUT   Y2
       LD    X1           ; 完成信号
       SET   S0           ; 回到初始步（循环）
       ; 也可以 SET S20 重新开始

RET                       ; 步进程序结束
END
```



------

#### 6. 步进指令的执行机制

| 当前激活步                           | 执行结果                                                   |
| :----------------------------------- | :--------------------------------------------------------- |
| S20=1, S21=0                         | 只执行STL S20内的程序，S21内的程序不执行                   |
| 当S20内的转移条件成立，执行`SET S21` | S20自动变为0（STL S20触点断开），S21变为1，开始执行STL S21 |

> 自动复位：前一状态的所有**OUT线圈**（Y, M, T）全部复位（定时器也会复位）。如果希望某个输出在多个步中保持，应使用`SET` / `RST`指令。

------

#### 7. 步进梯形图与普通梯形图的主要区别

| 项目     | 普通梯形图           | 步进梯形图（STL）                      |
| :------- | :------------------- | :------------------------------------- |
| 执行方式 | 每个周期全部扫描     | 只扫描当前激活的STL步                  |
| 输出保持 | 线圈保持上一扫描结果 | 步转移时，步内OUT线圈自动复位          |
| 程序结构 | 无特殊结构           | 必须用STL/RET划分步骤                  |
| 适用场合 | 所有逻辑控制         | 顺序流程控制（机械手、生产线、红绿灯） |

------

#### 8. 注意事项（重要）

1. **不能嵌套使用**
   一个STL步内不能再写另一个STL指令（即不能STL嵌套）。但可以通过`SET`转移到其他STL步。
2. **定时器冲突**
   相邻两个步最好不要使用同一个定时器编号。例如S20用T0，S21也用T0，当S20转移到S21时T0会自动复位，可能造成S21定时不准。
3. **转移条件建议用脉冲**
   使用`LD X0`直接转移可能会在一个扫描周期内连续多次转移。推荐使用`LDP X0`（上升沿）或`PLS M0`后再用`LD M0`。
4. **初始步必须被激活**
   上电后S0~S9不会自动激活，必须用`M8002`（初始化脉冲）或`ZRST`等方式设置。常用`SET S0`。
5. **RET必须放在最后**
   所有STL步结束后，必须写一条RET。RET之后不能再有STL指令。
6. **避免在STL步内使用MC/MCR**
   主控指令与步进指令混用可能导致逻辑混乱。

------

#### 9. 完整示例：红绿灯循环控制（使用步进指令）

**要求**：按下X20启动 → 红灯Y0亮10秒 → 黄灯Y1亮5秒 → 绿灯Y2亮10秒 → 循环。

plaintext

```
       LD     M8002        ; 上电初始脉冲
       SET    S0           ; 激活初始步

STL    S0                  ; 初始步：等待启动
       LD     X20
       SET    S10          ; 转红灯步

STL    S10                 ; 红灯步
       OUT    Y0           ; 红灯亮
       OUT    T0   K100    ; 10秒
       LD     T0
       SET    S11          ; 转黄灯步

STL    S11                 ; 黄灯步
       OUT    Y1
       OUT    T1   K50     ; 5秒
       LD     T1
       SET    S12          ; 转绿灯步

STL    S12                 ; 绿灯步
       OUT    Y2
       OUT    T2   K100    ; 10秒
       LD     T2
       SET    S10          ; 循环回红灯步

RET
END
```



**解释**：

- S10执行时，Y0亮，T0计时；时间到后SET S11，S10自动复位，Y0熄灭。
- 之后S11执行，Y1亮…以此类推。
- 绿灯结束后跳回S10，形成无限循环。

------

#### 10. 总结

| 指令 | 含义   | 用法                                              |
| :--- | :----- | :------------------------------------------------ |
| STL  | 步开始 | `STL S20` 定义第20步的开始                        |
| RET  | 步结束 | 所有STL步的最后写 `RET`                           |
| SET  | 转移   | `SET S21` 从当前步转移到S21（同时自动复位当前步） |

步进指令让顺序控制变得清晰、可靠、易于调试。掌握它，就能轻松编写各类流程控制程序。如果有具体工艺要求（如多分支、并行流程），还可以使用**选择分支**和**并行分支**（通过多个SET实现），这是步进指令的高级用法。







### 主控指令

**MC（Master Control）主控指令** 和 **MCR（Master Control Reset）主控复位指令** 是三菱PLC中用于实现**主控功能**的指令，它们可以成对使用，形成一个“主控区域”。其核心作用是根据一个公共条件（如一个触点）来决定是否执行某一段程序。

------

#### 1. 基本概念

- **MC**：主控指令，用于**开始**一个主控区域。当MC前面的条件成立时，该区域内的程序正常执行；当条件不成立时，该区域内的程序被**跳过**，且区域内的**非保持型线圈**（如普通输出Y、定时器T等）全部变为OFF，但保持型指令（如SET、RST、计数器等）不受影响。
- **MCR**：主控复位指令，用于**结束**一个主控区域。每个MC必须有一个对应的MCR。

**通俗理解**：MC相当于一个“总开关”，当它打开时，后面的电路（梯形图）才有电；当它关闭时，后面整个电路都被切断，里面的灯泡、接触器都会熄灭（复位）。

------

#### 2. 指令格式

- **MC N0 M100**
  - `N0`：嵌套等级，范围为N0~N7，允许主控指令嵌套使用（最多8层）。
  - `M100`：指定一个辅助继电器（或状态继电器），作为该主控的标志位（占用一个继电器号）。
- **MCR N0**
  - `N0`：嵌套等级，必须与对应的MC的等级一致。

------

#### 3. 工作原理

当MC前的触点导通时：

- 该MC指定的辅助继电器（如M100）得电。
- MC与对应MCR之间的程序**正常执行**，就像没有MC指令一样。

当MC前的触点断开时：

- M100失电。
- MC与MCR之间的所有程序被**跳过**（不执行），并且该区域内所有**非保持型线圈**（如OUT Y0、OUT T0等）自动变为OFF。
- **保持型指令**（SET、RST、应用指令等）不受影响，它们会保持原有状态。

------

#### 4. 简单示例

假设有一个绿灯Y0和定时器T0，用X0作为总开关控制其工作：

plaintext

```
    LD   X0        ; 总开关
    MC   N0  M0    ; 开始主控区域，等级0，标志继电器M0
    LD   X1        ; 子条件
    OUT  Y0        ; 绿灯
    OUT  T0  K100  ; 10秒定时器
    MCR  N0        ; 结束主控区域
```



- 当X0 = ON：MC条件满足，X1控制Y0和T0正常动作。
- 当X0 = OFF：整个MC~MCR之间的程序被跳过，Y0和T0（非保持型）都变为OFF，无论X1是ON还是OFF。

------

#### 5. 嵌套使用

主控指令可以嵌套，例如：

plaintext

```
    LD   X0
    MC   N0  M0     ; 第一层主控
    LD   X1
    MC   N1  M1     ; 第二层主控（嵌套在N0内）
    OUT  Y0
    MCR  N1         ; 结束第二层
    MCR  N0         ; 结束第一层
```



嵌套时等级必须从小到大，结束时要逆序（先内层后外层）。

------

#### 6. 与步进指令（STL/RET）的区别

| 特性     | MC/MCR（主控指令）                                       | STL/RET（步进指令）                                          |
| :------- | :------------------------------------------------------- | :----------------------------------------------------------- |
| 用途     | 根据一个公共条件跳过或执行一段程序                       | 实现顺序控制，按步骤转移                                     |
| 结构     | 一对指令包围一个区域                                     | 每个步（STL）是独立的一个段，最后用RET结束                   |
| 执行逻辑 | 条件成立时区域内程序正常执行；不成立时跳过，内部线圈复位 | 只有当前激活的步（状态）内的程序被执行，转移时自动复位前一步 |
| 保持性   | 区域内OUT线圈在条件断开时自动复位                        | 步转移时，前一步的OUT线圈自动复位（但SET/RST保持）           |
| 适用场景 | 用于“总开关”控制多个输出                                 | 用于流程分步控制，如红绿灯、机械手动作                       |

------

#### 7. 注意事项

- MC和MCR必须成对使用，且嵌套等级必须匹配。
- MC指定的辅助继电器（如M0）不能与其它线圈重复使用。
- 在MC~MCR区域内可以继续使用其他指令，但不能使用END、STL等指令。
- 由于MC/MCR会使区域内OUT线圈在条件断开时强制复位，可能造成设备意外停止，使用时需谨慎。

掌握MC/MCR有助于简化一些需要公共条件控制的多段程序，但现代编程中常用更直观的“分支”结构（如使用辅助继电器与常开触点串联）来替代，以避免主控带来的“隐性”复位问题。



## 高速计数器-1相1输入-1相2输入-2相2输入



### 1. 1相1输入（单相单输入）

- **特点**：只有一个脉冲输入端子（如X0、X1等），计数器仅对该端子输入的脉冲进行**增计数**（或固定方向）。
- **方向控制**：方向不可变，只能单向计数；部分机型可通过特殊辅助继电器（如M8235）强制改变计数方向，但本质上仍为单输入。
- **典型计数器**：C235~C245（FX3U中）。
- **应用**：适用于不需要改变方向、只累计脉冲数量的场合，如圈数计数、长度测量（单向）。

**示例**：C235使用X0输入，每来一个脉冲，当前值加1。

------

### 2. 1相2输入（单相双输入）

- **特点**：有两个脉冲输入端子：一个**增计数输入**（如X0），一个**减计数输入**（如X1）。
  增输入端子收到脉冲时，当前值**加1**；减输入端子收到脉冲时，当前值**减1**。
- **方向控制**：由哪个端子有脉冲决定，无需额外方向信号。
- **典型计数器**：C246~C250（FX3U中）。
- **应用**：适用于可正反转的编码器（输出两路脉冲，分别代表正向和反向脉冲），或需要双向计数的场合，如卷绕机、双向传送带。

**示例**：C246使用X0（增）、X1（减）。当电机正转时，X0发脉冲，计数增加；反转时，X1发脉冲，计数减少。

------

### 3. 对比表

| 项目             | 1相1输入                 | 1相2输入                         |
| :--------------- | :----------------------- | :------------------------------- |
| 输入端子数       | 1个（脉冲）              | 2个（增脉冲 + 减脉冲）           |
| 计数方向         | 固定（或通过软元件改变） | 由哪个输入端有脉冲决定           |
| 典型计数器编号   | C235~C245                | C246~C250                        |
| 可实现的计数模式 | 单向增/减                | 双向增减                         |
| 与编码器匹配     | 单输出编码器             | 正反转脉冲输出编码器（如推挽式） |

------

### 4. 补充说明

- 还有**2相2输入**（AB相模式），通过A、B两相信号的相位差自动判别方向，支持4倍频，适用于正交编码器。
- 使用时需注意高速计数器的输入端子是固定的（如X0~X7），不能随意映射。
- 程序中需通过`DHSCS`、`DHSCR`等指令进行高速比较和中断处理。

理解这两种模式有助于正确选择计数器类型，匹配现场传感器或编码器的输出信号。



## 2相2输入

**2相2输入**是三菱FX系列PLC（如FX3U）高速计数器的一种工作模式，通常用于连接**正交编码器**（A相、B相输出）。它通过两路脉冲的**相位差**来自动判别旋转方向，并实现高精度的位置或速度检测。

------

### 1. 基本原理

- **输入信号**：需要两个输入端子，分别接入编码器的 **A相** 和 **B相** 脉冲信号。
- **方向判别**：
  - 若A相 **超前** B相 90°，计数器**加计数**（正转）。
  - 若B相 **超前** A相 90°，计数器**减计数**（反转）。
- **计数倍率**：可设置为 **1倍频**、**2倍频** 或 **4倍频**（FX3U中通过指令或特殊寄存器选择）。
  - 1倍频：仅对A相上升沿计数。
  - 2倍频：对A相上升沿和下降沿计数。
  - 4倍频：对A、B两相的上升沿和下降沿都计数（分辨率最高，常用）。

------

### 2. 典型计数器编号

FX3U中支持2相2输入的计数器包括：

- **C251**：硬件计数比较（32位，双向）
- **C252**、**C253**、**C254**、**C255**（不同输入端子分配）

输入端子固定（如C251使用X0（A相）、X1（B相）），具体请查阅PLC手册。

------

### 3. 与1相2输入的区别

| 项目       | 1相2输入                     | 2相2输入                       |
| :--------- | :--------------------------- | :----------------------------- |
| 输入端子数 | 2个（增脉冲 + 减脉冲）       | 2个（A相 + B相）               |
| 方向信号   | 由脉冲来自哪个端子决定       | 由A/B相的相位差决定            |
| 典型编码器 | 正反转脉冲输出型（如推挽式） | 正交编码器（增量式）           |
| 最高倍频   | 1倍频（不能提高分辨率）      | 支持1、2、4倍频（4倍频最常用） |
| 抗干扰能力 | 一般                         | 较强（差分信号更好）           |
| 典型应用   | 简易双向计数                 | 伺服电机位置反馈、高精度定位   |

------

### 4. 应用示例

假设使用C251（A相→X0，B相→X1），并启用4倍频计数：

plaintext

```
LD    M8000       ; 运行时常通
OUT   C251        ; 启动C251高速计数（无需驱动线圈，由硬件直接计数）
```



程序中可通过 `DHSCS`（高速比较置位）等指令与当前值比较，实现中断或输出。

------

### 5. 注意事项

- 输入端子需支持高速响应（X0~X7，最高频率取决于型号，FX3U可达100kHz）。
- 编码器输出类型应为集电极开路或差分（推荐差分以提高抗干扰）。
- 若需4倍频，需设置特殊寄存器（如M8198等）或使用指令配置。

**总结**：2相2输入是最高精度、功能最全的高速计数模式，适合需要方向识别和高分辨率的运动控制场合。



# CJ-**Conditional Jump，条件跳转**

## 1. CJ指令的功能

**CJ** 用于跳过程序中的某一段，直接跳转到指定的**指针P**处继续执行。

- 当 CJ 指令前的条件为 **ON** 时，执行跳转，跳过 CJ 与目标指针之间的程序。
- 当条件为 **OFF** 时，不跳转，程序按顺序执行。

这样可以减少扫描时间、实现程序选择性执行或手动/自动切换等功能。

> **指针P**：程序标号，范围 P0~P4095（FX3U），用于标记跳转目标位置。

------

## 2. 指令格式

plaintext

```
CJ   P0      ; 跳转到标号 P0 处
```



或者

plaintext

```
CJ   P0      ; 当条件满足时跳转
```



------

## 3. 使用示例

### 例1：无条件跳转（常ON）

plaintext

```
LD   M8000      ; 运行监视常ON
CJ   P10        ; 始终跳转到P10
...             ; 这段程序永远不会执行
P10
...             ; 跳转后执行这里的程序
```



### 例2：条件跳转

plaintext

```
LD   X0         ; 当X0=ON时
CJ   P20        ; 跳过一段程序
LD   X1
OUT  Y0         ; X0=OFF时才执行
...
P20
LD   X2
OUT  Y1         ; 跳转后执行
```



------

## 4. 指针P的用法

指针单独占一行，格式为 `P0` 或 `P20` 等，后面紧跟程序。

plaintext

```
     LD   X0
     CJ   P99
     ...         ; 被跳过的程序
P99
     LD   X1
     OUT  Y0
```



**注意**：

- 同一指针编号只能在程序中出现**一次**。
- 指针可以放在程序任何位置（通常放在END前或子程序后）。
- 指针也可以作为子程序（CALL）的入口。

------

## 5. CJ与CALL的区别

| 指令 | 功能             | 返回方式                           |
| :--- | :--------------- | :--------------------------------- |
| CJ   | 条件跳转，不返回 | 继续顺序执行                       |
| CALL | 条件调用子程序   | 执行完子程序后返回CALL的下一条指令 |

**简单说**：CJ是“跳过去就不再回来”，CALL是“去子程序干完活再回来”。

------

## 6. 注意事项（非常重要）

1. **定时器/计数器在跳过期间**
   - 被跳过的程序段中的**定时器（T）** 会**暂停计时**（不执行OUT T指令，当前值不变）。
   - 跳过结束后再次执行时，定时器继续计时（但总时间会拉长）。
   - **计数器（C）** 在跳过期间不计数（不执行OUT C指令）。
   - **高速计数器**（硬件计数）不受CJ影响，始终计数。
2. **输出线圈状态**
   - 被跳过的程序段中的 `OUT Y`、`OUT M` 等线圈会**保持跳转前的状态**（不刷新）。
   - 因此可能造成输出“卡死”在某个状态，需谨慎使用。
3. **不能跳转到**
   - 跳转到 **STL** 步内（会导致步进混乱）。
   - 跨 **MC/MCR** 主控区域（必须成对匹配）。
   - 跳转到 **FOR/NEXT** 循环内部（会造成循环结构破坏）。
   - 跳转到 **中断程序** 或 **子程序** 内（应使用CALL）。
4. **指针位置**
   - 指针不能放在 `FOR ~ NEXT` 内部，也不能放在 `STL ~ RET` 内部。
   - 通常将指针放在程序末尾、`END` 之前，或单独一个程序块中。
5. **多个CJ共用同一指针**
   - 可以，只要条件逻辑正确。
   - 但不可出现两个相同编号的指针。

------

## 7. 典型应用场景

- **手动/自动模式切换**：通过CJ跳过自动程序或手动程序段。
- **调试与测试**：临时跳过某些故障或未完成的程序段。
- **提高扫描速度**：在特定条件下跳过不需要执行的程序块。
- **多工艺选择**：根据产品型号选择执行不同的控制流程。

------

## 8. 程序示例：手动/自动切换

plaintext

```
     LD   M0        ; M0=1 自动模式，M0=0 手动模式
     CJ   P10       ; 若M0=0，跳转到手动程序段
     
     // 自动程序段
     LD   X0
     OUT  Y0
     ... 
     CJ   P20       ; 跳过手动程序段
     
P10                 ; 手动程序段
     LD   X1
     OUT  Y0
     ...
     
P20                 ; 公共程序
     LD   X2
     OUT  Y1
     END
```



------

## 9. 总结

| 指令 | 格式    | 功能                   | 跳转后           |
| :--- | :------ | :--------------------- | :--------------- |
| CJ   | `CJ Pn` | 条件满足时跳转到指针Pn | 不返回，继续执行 |
| 指针 | `Pn`    | 跳转目标标号           | 作为程序分段标记 |

**记住**：CJ跳过的程序中的定时器会暂停、输出会保持，这是它最容易被忽略的特性。合理使用CJ可以简化程序结构，但滥用会使程序难以调试。现代编程中，建议多用**辅助继电器 + 分支**替代简单的CJ跳转。



# **CALL（子程序调用）**

### 1. 基本概念

- **CALL**：条件调用一个子程序（由指针Pxx标识），执行完子程序后通过 **SRET** 返回到 CALL 指令的下一条指令，继续执行主程序。
- **子程序**：独立编写的一段程序，以 `Pxx` 为入口，以 `SRET` 结束，可被主程序或其它子程序多次调用。
- **指针P**：子程序的标号，范围 P0~P4095（FX3U）。同一指针编号只能出现一次。

> 可以把子程序理解为一个 **“可以随时被叫去干活，干完就回来”** 的小程序块。

------

### 2. 指令格式

#### 调用子程序

plaintext

```
CALL  P10      ; 调用以P10为入口的子程序
```



条件调用（常用）：

plaintext

```
LD   X0
CALL  P20      ; 当X0=ON时调用P20子程序
```



#### 子程序结构

plaintext

```
P10            ; 子程序入口
    ...        ; 子程序逻辑
    SRET       ; 子程序返回
```



------

### 3. 完整示例

**主程序**：当X0接通时，调用子程序P10，执行完后继续执行后面的程序。

plaintext

```
        LD     M8000      ; 常ON
        OUT    Y0         ; 主程序点亮Y0
        LD     X0
        CALL   P10        ; 调用子程序P10
        LD     X1
        OUT    Y1         ; 从子程序返回后执行
        END               ; 主程序结束

P10                      ; 子程序入口
        LD     X2
        OUT    Y2
        SRET              ; 返回主程序
```



**执行流程**：

- 主程序扫描，Y0一直亮。
- 若X0=ON，则跳转到P10，执行Y2控制逻辑，然后SRET返回，继续执行X1控制Y1。
- 若X0=OFF，则跳过CALL，直接执行X1控制Y1。

------

### 4. 子程序的放置位置

子程序必须写在 **FEND（主程序结束）指令之后，END指令之前**。
标准程序结构：

plaintext

```
    0   ...  主程序（包括CALL指令）
        FEND               ; 主程序结束
    ...  子程序1
    ...  子程序2
        END                ; 整个程序结束
```



- **FEND**：表示主程序结束，PLC执行完主程序后不再往下扫描，但会为子程序保留空间。
- 如果主程序后没有FEND，CPU会继续执行子程序区，造成逻辑错误。

------

### 5. 嵌套调用

子程序内部可以再调用其它子程序，称为**嵌套调用**。FX3U允许最多 **16级** 嵌套。

plaintext

```
主程序:
    CALL P10
    ...
P10:
    CALL P20
    SRET
P20:
    SRET
```



注意：嵌套过深会消耗更多堆栈空间，可能导致栈溢出（PLC会报错）。

------

### 6. CALL 与 CJ 的区别

| 特性       | CALL                   | CJ                         |
| :--------- | :--------------------- | :------------------------- |
| 功能       | 调用子程序，执行完返回 | 条件跳转到指定标号，不返回 |
| 返回指令   | SRET                   | 无（顺序执行）             |
| 目标       | 子程序入口（Pxx）      | 程序标号（Pxx）            |
| 堆栈操作   | 压栈返回地址           | 无                         |
| 适用场景   | 重复使用的功能模块     | 选择性跳过一段程序         |
| 定时器行为 | 正常计时（不被暂停）   | 被跳过的定时器暂停         |

**简单记忆**：

- CALL：去子程序干活，干完**回来**。
- CJ：跳过去，**不再回来**。

------

### 7. 注意事项

1. **子程序不能调用自己**（递归调用不允许），否则会导致堆栈溢出。
2. **子程序中不要使用 END 指令**，必须用 SRET 返回。
3. **子程序内可以使用 MC/MCR**，但要确保成对出现，且不能跨子程序边界。
4. **子程序内可以使用 CJ 跳转**，但只能跳到同一个子程序内的标号，不能跳到主程序或其他子程序。
5. **子程序内的定时器、计数器** 与主程序一样正常计时/计数（不会被CJ影响）。
6. **子程序也可以被中断程序调用**（但需注意中断嵌套深度）。
7. **多个CALL可以共用同一个子程序**（代码复用，节省内存）。

------

### 8. 典型应用场景

- **重复运算**：如PID计算、转换公式，在主程序多处调用。
- **标准化功能**：如报警处理、显示刷新，独立成子程序。
- **手动/自动切换**：分别写成子程序，根据模式调用。
- **简化主程序**：将复杂逻辑拆分为多个子程序，提高可读性。

------

### 9. 总结

| 指令    | 功能         | 出现位置               |
| :------ | :----------- | :--------------------- |
| CALL Pn | 调用子程序Pn | 主程序或子程序内       |
| SRET    | 从子程序返回 | 每个子程序的末尾       |
| FEND    | 主程序结束   | 主程序之后、子程序之前 |
| END     | 程序结束     | 所有程序块之后         |

使用 CALL 指令可以让程序模块化、结构清晰、易于调试和维护。掌握它与 CJ 的区别，能更好地设计PLC程序流程。

FEND 主程序结束指令

SRET 子程序结束指令

EI 中断允许指令

ID 中断禁止指令

P 指针P

I 外部中断



# **EI（Enable Interrupt，允许中断）指令**

### 1. EI 指令的基本作用

- **EI**：全局允许中断。执行EI后，PLC响应已定义的中断事件（输入中断、定时器中断、高速计数器中断等），并执行相应的中断程序。
- 与之对应的 **DI（Disable Interrupt）** 指令用于禁止中断。
- 默认情况下，PLC上电后中断是**禁止**的，必须用EI指令开启。

> 可以把中断理解成：主程序按顺序执行时，突然插进来一个“紧急任务”（中断程序），处理完后再回到主程序继续执行。

------

### 2. 中断的类型（FX3U为例）

| 中断类型       | 指针编号  | 触发条件                 | 典型用途         |
| :------------- | :-------- | :----------------------- | :--------------- |
| 输入中断       | I0□~I5□   | X0~X5的上升沿/下降沿     | 高速响应输入信号 |
| 定时器中断     | I6□□~I8□□ | 定时时间到（0.5~1000ms） | 周期执行任务     |
| 高速计数器中断 | I010~I060 | 计数当前值与设定值一致   | 精确定位         |

**输入中断指针格式**：`I 0 0 0`

- 第一位：0~5 对应 X0~X5
- 第二位：0=下降沿，1=上升沿
- 第三位：固定为0
  例如：`I001` = X0上升沿中断；`I100` = X1下降沿中断。

**定时器中断**：`I6 30`（每30ms执行一次），范围 I6□□~I8□□（10~99，步长1ms？实际为0.5~1000ms，FX3U支持最多3个定时器中断）。

------

### 3. 中断程序的结构

中断程序必须写在 **主程序结束（FEND）之后，END之前**，以中断指针为入口，以 **IRET（中断返回）** 结束。

plaintext

```
       LD     M8000      ; 常ON
       EI                ; 允许中断
       ...               ; 主程序
       FEND              ; 主程序结束

I001                    ; X0上升沿中断程序
       INC    D0
       IRET               ; 中断返回

I100                    ; X1下降沿中断程序
       OUT    Y0
       IRET

END
```



------

### 4. EI / DI 的使用规则

- **EI 和 DI 成对使用**：在主程序中用EI开启中断，在不需要中断的地方用DI关闭。
- **中断嵌套**：FX3U支持中断优先级（输入中断优先于定时器中断，同一类型按指针号顺序），高优先级中断可打断低优先级中断程序。
- **中断程序中不能使用的指令**：
  - EI / DI（允许/禁止中断本身）
  - CJ（跳转）
  - STL / RET（步进指令）
  - FOR / NEXT（循环）
  - CALL / SRET（子程序调用）
  - MC / MCR（主控指令）
    因为这些指令会破坏中断的返回机制。
- **中断程序应尽量简短**，避免长时间占用CPU，否则会影响主程序扫描周期。

------

### 5. 典型应用示例

#### 例1：输入中断（X0上升沿触发，D0加1）

plaintext

```
       LD     M8002      ; 上电初始脉冲
       EI                ; 允许中断
       ...               ; 主程序
       FEND

I001                    ; X0上升沿中断
       INC    D0
       IRET

END
```



#### 例2：定时器中断（每10ms执行一次）

plaintext

```
       LD     M8000
       EI
       ...               ; 主程序
       FEND

I610                   ; 10ms定时器中断（I6 10）
       INC    D10
       IRET

END
```



**注意**：定时器中断需要在程序中使用 `MOV` 等指令设定时间，或者直接通过参数设置。FX3U中定时器中断的编号规则：I6□□，其中□□为时间（10~99，单位ms）。

------

### 6. 注意事项总结

1. **EI 必须放在主程序中**，不能放在中断程序内。
2. **中断指针不能重复**，同一指针只能定义一次。
3. **中断程序必须以 IRET 结束**，否则PLC报错。
4. **中断响应时间**：输入中断延迟约为几十微秒到几百微秒，具体取决于PLC型号和当前扫描周期。
5. **中断过多或过长**会导致主程序扫描周期变长，甚至看门狗超时（WDT错误）。
6. **某些指令在中断中无效**（如通讯指令、PID指令等），使用前请查阅手册。
7. **DI 指令**可以暂时屏蔽中断，例如在关键数据读写时禁止中断，防止数据不一致。

------

### 7. 总结

| 指令     | 功能                   | 位置               |
| :------- | :--------------------- | :----------------- |
| EI       | 允许全局中断           | 主程序（FEND之前） |
| DI       | 禁止全局中断           | 主程序             |
| 中断指针 | 中断程序入口（如I001） | FEND之后           |
| IRET     | 中断返回               | 每个中断程序的末尾 |

**一句话记忆**：`EI` 打开中断大门，`IRET` 从紧急任务返回，`DI` 把大门关上。掌握中断能实现高速、实时的控制响应，是高级PLC编程的重要技巧。



# **CMP（比较）指令**

用于比较两个数值的大小，并将结果输出到指定的位元件（通常为M、Y等）。

------

### 1. 指令格式

**CMP** 是比较指令，基本格式为：

plaintext

```
CMP   S1   S2   D
```



- **S1**：比较值1（常数 K/H 或数据寄存器 D 等）
- **S2**：比较值2（常数 K/H 或数据寄存器 D 等）
- **D**：比较结果的起始位元件（占用连续 3 个位）

### 示例

plaintext

```
CMP   D0   D10   M0
```



比较 D0 和 D10 的大小，结果影响 M0、M1、M2。

------

### 2. 比较结果

执行 CMP 后，根据 S1 与 S2 的关系，将 **D**、**D+1**、**D+2** 三个位元件置为 ON/OFF：

| 条件     | D    | D+1  | D+2  |
| :------- | :--- | :--- | :--- |
| S1 ＜ S2 | ON   | OFF  | OFF  |
| S1 ＝ S2 | OFF  | ON   | OFF  |
| S1 ＞ S2 | OFF  | OFF  | ON   |

以上述 `CMP D0 D10 M0` 为例：

- 若 D0 < D10，则 M0=ON，M1=OFF，M2=OFF；
- 若 D0 = D10，则 M0=OFF，M1=ON，M2=OFF；
- 若 D0 > D10，则 M0=OFF，M1=OFF，M2=ON。

------

### 3. 数据类型

- **16位比较**：CMP（K、D 等为 16位，范围 -32768~32767）
- **32位比较**：DCMP（操作数为 32位，如 D0 与 D1 组成 32位，或直接用 K 超出 16位范围）

示例：

plaintext

```
DCMP   D0   D10   M10    ; 比较 (D1,D0) 与 (D11,D10)
```



------

### 4. 使用注意事项

1. **目标位元件连续占用 3 个**
   如 `CMP D0 D10 M0` 会占用 M0、M1、M2，编程时注意不要在其他地方重复使用这些位（除非特意复用）。

2. **比较指令最好用脉冲执行**
   直接使用 CMP（连续执行型）会在每个扫描周期都进行比较并刷新结果，若只需要在条件变化时比较，推荐使用 **CMPP**（脉冲型）。

   例如：

   plaintext

   ```
   LD    X0
   CMPP  D0 D10 M0   ; 仅在 X0 上升沿时执行一次比较
   ```

   

3. **避免出现“等于”区间误判**
   当数值频繁变化时，三个位可能会瞬间跳动。如果需要稳定区间判断，可以使用 **ZCP（区间比较）** 指令。

4. **比较数据范围一致**
   16位比较时操作数不能超过 16位范围，否则应使用 DCMP。

5. **浮点数比较**
   需要浮点数比较时，使用 `DECMP`（FX3U 支持浮点数）。

------

### 5. 完整示例

**要求**：当 X0 按下时，比较 D0 与 D10 的值，若 D0 > D10 则 Y0 亮；若等于则 Y1 亮；若小于则 Y2 亮。

plaintext

```
        LD      X0
        CMPP    D0 D10 M0   ; 脉冲执行，只比较一次

        LD      M2          ; D0 > D10
        OUT     Y0

        LD      M1          ; D0 = D10
        OUT     Y1

        LD      M0          ; D0 < D10
        OUT     Y2
```



------

### 6. 区间比较指令 ZCP（扩展）

**ZCP** 用于判断一个数值是否落在指定的下限与上限之间（闭区间）。

格式：

plaintext

```
ZCP   S1   S2   S3   D
```



- S1：下限值
- S2：上限值（必须 ≥ S1）
- S3：比较值
- D：结果起始位（占用 3 个位）

结果：

| 条件         | D    | D+1  | D+2  |
| :----------- | :--- | :--- | :--- |
| S3 ＜ S1     | ON   | OFF  | OFF  |
| S1 ≤ S3 ≤ S2 | OFF  | ON   | OFF  |
| S3 ＞ S2     | OFF  | OFF  | ON   |

示例：

plaintext

```
ZCP   K10   K100   D0   M10
```



判断 D0 是否在 [10, 100] 内，结果送 M10~M12。

------

### 7. 总结

| 指令  | 功能                  | 占用位点数 | 常用场景             |
| :---- | :-------------------- | :--------- | :------------------- |
| CMP   | 两个数值比较（<,=,>） | 3          | 压力/温度上下限判断  |
| ZCP   | 一个数值是否在区间内  | 3          | 工艺参数范围监控     |
| DCMP  | 32位数值比较          | 3          | 高速计数器当前值比较 |
| DECMP | 浮点数比较            | 3          | 高精度模拟量处理     |

**核心技巧**：配合脉冲执行（CMPP）避免每周期刷新，注意目标位元件的独占使用，就能灵活实现各种比较逻辑。

# **ZCP（区间比较）指令**-**Zone Compare**



它是CMP指令的扩展，用于判断一个数值是否落在指定的下限值与上限值之间（闭区间）。

------

### 1. 指令格式

**ZCP** 的基本格式：

plaintext

```
ZCP   S1   S2   S3   D
```



- **S1**：区间下限值（常数 K/H 或数据寄存器 D 等）
- **S2**：区间上限值（常数 K/H 或数据寄存器 D 等），必须满足 S1 ≤ S2
- **S3**：待比较的数值（常数或数据寄存器）
- **D**：比较结果的起始位元件（占用连续 **3** 个位）

### 示例

plaintext

```
ZCP   K10   K100   D0   M0
```



判断 D0 的值是否在 [10, 100] 区间内，结果影响 M0、M1、M2。

------

### 2. 比较结果

执行 ZCP 后，根据 S3 与区间 [S1, S2] 的关系，将 **D**、**D+1**、**D+2** 三个位元件置为 ON/OFF：

| 条件         | D    | D+1  | D+2  |
| :----------- | :--- | :--- | :--- |
| S3 ＜ S1     | ON   | OFF  | OFF  |
| S1 ≤ S3 ≤ S2 | OFF  | ON   | OFF  |
| S3 ＞ S2     | OFF  | OFF  | ON   |

以上述 `ZCP K10 K100 D0 M0` 为例：

- 若 D0 < 10，则 M0=ON，M1=OFF，M2=OFF；
- 若 10 ≤ D0 ≤ 100，则 M0=OFF，M1=ON，M2=OFF；
- 若 D0 > 100，则 M0=OFF，M1=OFF，M2=ON。

------

### 3. 数据类型

- **16位比较**：ZCP（操作数为 16位，范围 -32768~32767）
- **32位比较**：DZCP（操作数为 32位，如 D0 与 D1 组成 32位）

示例：

plaintext

```
DZCP  D0   D2   D4   M10   ; 比较 (D5,D4) 是否在 [(D1,D0), (D3,D2)] 区间内
```



注意：使用 DZCP 时，S1、S2、S3 都要占用连续两个寄存器（低位在前）。

------

### 4. 使用注意事项

1. **S1 必须 ≤ S2**
   如果程序中 S1 > S2，PLC 会将 S2 视为下限、S1 视为上限（自动交换），但为了程序清晰，应保证下限 ≤ 上限。

2. **目标位元件连续占用 3 个**
   如 `ZCP K10 K100 D0 M0` 会占用 M0、M1、M2。编程时注意不要在其他地方重复使用这些位（除非特意复用）。

3. **脉冲执行推荐**
   直接使用 ZCP（连续执行型）会在每个扫描周期都进行比较并刷新结果。若只需要在条件变化时比较，推荐使用 **ZCPP**（脉冲型）。

   例如：

   plaintext

   ```
   LD    X0
   ZCPP  K10 K100 D0 M0   ; 仅在 X0 上升沿时执行一次比较
   ```

   

4. **区间边界包含**
   ZCP 是闭区间（包含端点）。如果需要开区间，可以手动调整边界值（如 S1+1、S2-1）。

5. **浮点数区间比较**
   FX3U 支持 `DEZCP` 指令进行浮点数区间比较。

------

### 5. 完整示例

**要求**：检测温度值 D0，若低于 10℃ 则开启加热器 Y0；若在 10℃~100℃ 之间则正常运行 Y1；若高于 100℃ 则报警 Y2。

plaintext

```
        LD      M8000      ; 常ON
        ZCP     K10 K100 D0 M0   ; 区间比较，结果存 M0~M2

        LD      M0          ; D0 < 10
        OUT     Y0          ; 加热

        LD      M1          ; 10 ≤ D0 ≤ 100
        OUT     Y1          ; 正常

        LD      M2          ; D0 > 100
        OUT     Y2          ; 报警
```



如果需要仅在 D0 变化时比较，可将 `LD M8000` 改为 `LD X0` 并配合 `ZCPP`。

------

### 6. ZCP 与 CMP 的区别

| 指令 | 比较方式              | 输出结果                                   | 应用场景                         |
| :--- | :-------------------- | :----------------------------------------- | :------------------------------- |
| CMP  | 两个数值之间（<,=,>） | 三个位分别表示小于、等于、大于             | 单一阈值判断（如是否超限）       |
| ZCP  | 一个数值是否在区间内  | 三个位分别表示低于下限、在区间内、高于上限 | 区间监控（如温度、压力正常范围） |

**简单记忆**：

- CMP：比大小（和单一标准比）
- ZCP：看是否在范围内（和上下限比）

------

### 7. 常见错误

1. **下限 > 上限**
   虽然 PLC 会自动交换，但会降低程序可读性，且可能导致意外结果（若上下限来自变化的数据，交换后可能不符合工艺要求）。
2. **目标位被其他指令覆盖**
   例如 ZCP 使用了 M0~M2，后面又用 OUT M1 直接赋值，会破坏比较结果。建议专门留出一段连续的位用于 ZCP。
3. **忘记使用脉冲执行**
   如果 ZCP 放在每个周期都执行的位置，且后续逻辑依赖比较结果的上升沿，可能会误触发。此时应改用 ZCPP。

------

### 8. 总结

| 指令 | 功能                 | 占用位点数 | 常用场景                       |
| :--- | :------------------- | :--------- | :----------------------------- |
| ZCP  | 判断数值是否在区间内 | 3          | 模拟量监控（温度、压力、速度） |
| DZCP | 32位数值区间比较     | 3          | 高速计数器当前位置区间判断     |
| ZCPP | 脉冲执行版           | 3          | 只在条件触发时比较一次         |

**核心要点**：ZCP 让区间判断变得非常简单，只需一条指令就能得到三个区间的结果，配合脉冲执行可大幅简化梯形图。



# `K2Y0` 的详细解析

- **K2** → 2 个 4 位组 → 8 位数据。
- **Y0** → 起始位地址。

因此，`K2Y0` 占用 **Y0、Y1、Y2、Y3、Y4、Y5、Y6、Y7** 共 8 个输出点。
其内部位排列（高位→低位）为：`Y7 Y6 Y5 Y4 Y3 Y2 Y1 Y0`。

> 注意：三菱PLC中输出编号为**八进制**，Y0~Y7 之后是 Y10，但 `K2Y0` 只占用 Y0~Y7，不会跨到 Y10。







# **MOV（传送）指令**

这是最基础、最常用的数据指令之一。

------

#### 1. 指令格式

**MOV** 是数据传送指令，基本格式为：

plaintext

```
MOV   S   D
```



- **S**：源数据（常数 K/H 或数据寄存器 D、T、C 当前值等）
- **D**：目标元件（数据寄存器 D、T、C 当前值、位组合元件等）

### 示例

plaintext

```
MOV   K100   D0      ; 将常数 100 传送到 D0
MOV   D0     D10     ; 将 D0 的值传送到 D10
MOV   T0     D20     ; 将定时器 T0 的当前值传送到 D20
```



------

#### 2. 功能说明

MOV 指令将源数据 **原样复制** 到目标元件中。源数据本身不变，目标元件原来的内容被覆盖。

- **16位传送**：MOV 处理 16位数据（-32768~32767），占用一个数据寄存器。
- **32位传送**：**DMOV** 处理 32位数据（-2147483648~2147483647），占用两个连续的数据寄存器。

### 32位示例

plaintext

```
DMOV   K100000   D0    ; 将 100000 传送到 (D1,D0) 组成的 32位寄存器
```



------

#### 3. 适用元件

| 元件类型             | 可作为 S | 可作为 D                       |
| :------------------- | :------- | :----------------------------- |
| 常数（K/H）          | ✅        | ❌                              |
| 数据寄存器 D         | ✅        | ✅                              |
| 定时器 T 当前值      | ✅        | ✅（但注意 T 是只读？实际可写） |
| 计数器 C 当前值      | ✅        | ✅                              |
| 位组合（K1M0等）     | ✅        | ✅                              |
| 特殊寄存器（D8000~） | ✅        | ✅                              |

**位组合示例**：

plaintext

```
MOV   K1X0   D0     ; 将 X0~X3 组成的4位数据传送到 D0 的低4位
MOV   D0     K1Y0   ; 将 D0 的低4位传送到 Y0~Y3
```



------

#### 4. 脉冲执行与连续执行

- **连续执行型（MOV）**：每个扫描周期都执行传送。
  若传送条件一直为 ON，则每个周期都重复传送（效率较低，但常用于需要实时更新的数据）。
- **脉冲执行型（MOVP）**：仅在条件从 OFF→ON 的上升沿执行一次传送。
  推荐使用脉冲执行，避免不必要的重复操作。

plaintext

```
LD    X0
MOVP  K100  D0     ; 仅在 X0 按下瞬间传送一次
```



------

#### 5. 常用场景

1. **初始化数据**：上电时将预设值送入数据寄存器。

   plaintext

   ```
   LD    M8002      ; 上电初始脉冲
   MOV   K0    D0   ; 清零 D0
   MOV   K100  D10  ; 设置定时器预设值
   ```

   

2. **数值运算准备**：将参与运算的数据移到指定寄存器。

3. **状态暂存**：保存当前值，例如：

   plaintext

   ```
   MOV   D0    D100   ; 备份 D0 的值
   ```

   

4. **输出位组合**：用 MOV 一次性控制多个 Y 输出。

   plaintext

   ```
   MOV   K5    K2Y0   ; 将二进制 0101 输出到 Y0~Y3（Y0=1,Y1=0,Y2=1,Y3=0）
   ```

   

5. **读取高速计数器当前值**：

   plaintext

   ```
   DMOV  C251  D0     ; 将32位计数器C251的当前值读到 D1,D0
   ```

   

------

#### 6. 注意事项

1. **数据类型匹配**
   - 16位传送不能超过 32767，否则高位丢失。
   - 32位数据必须用 DMOV，否则只传送低16位。
2. **目标元件范围**
   不要超出有效地址范围（如 D0~D7999 通用，D8000~D8511 为特殊寄存器，只读或具有特定功能）。
3. **位组合的位数**
   `K1Y0` 表示从 Y0 开始的4位（1个nibble），`K2Y0` 表示8位（1个字节），`K4Y0` 表示16位。
   传送时数据长度必须匹配，否则高位会被截断或填充0。
4. **MOV 与 DMOV 对元件占用的影响**
   使用 `DMOV D0 D10` 时，D10 和 D11 都被占用。如果后续程序使用了 D11，需注意可能被覆盖。
5. **禁止在中断程序与主程序间同时读写同一数据寄存器**
   可能造成数据不一致，必要时使用 DI 暂时关闭中断。

------

#### 7. 扩展：SMOV（移位传送）

三菱FX还有 **SMOV** 指令，用于按位（十进制位）移动并传送，常用于 BCD 码处理。格式：

plaintext

```
SMOV  S   m1   m2   D   n
```



例如将 S 的第 m1 位开始的 m2 位，传送到 D 的第 n 位开始的位置。

但 SMOV 使用较少，初学者先掌握 MOV 即可。

------

#### 8. 总结

| 指令 | 功能     | 数据长度 | 执行方式         |
| :--- | :------- | :------- | :--------------- |
| MOV  | 16位传送 | 16位     | 连续/脉冲(MOVP)  |
| DMOV | 32位传送 | 32位     | 连续/脉冲(DMOVP) |

**一句话记忆**：`MOV` 就是复制粘贴数据，`DMOV` 复制双倍数据。配合脉冲执行（加P）可以精确控制传送时机，是PLC数据操作的基础。



# **CML（反转传送）指令**-Complement Move



它是 MOV 指令的变种，在传送数据的同时将源数据的每一位取反（0→1，1→0）。

------

#### 1. 指令格式

**CML** 的基本格式：

plaintext

```
CML   S   D
```



- **S**：源数据（常数 K/H、数据寄存器 D、T/C 当前值、位组合等）
- **D**：目标元件（数据寄存器 D、位组合元件等）

### 示例

plaintext

```
CML   D0   D10     ; 将 D0 按位取反后传送到 D10
CML   K5   K2Y0    ; 将常数5取反后输出到 Y0~Y7
```



------

#### 2. 功能说明

CML 的执行过程分为两步：

1. 读取源数据 S 的每一位。
2. 将每一位 **取反（逻辑非）**，然后写入目标 D。

**真值表**：

| 源位 | 目标位 |
| :--- | :----- |
| 0    | 1      |
| 1    | 0      |

例如，S = 5（二进制 `0101`），取反后得到 `1010`（十进制 10），然后传送到 D。

------

#### 3. 数据类型与指令变体

- **16位反转传送**：`CML`（处理16位数据）
- **32位反转传送**：`DCML`（处理32位数据）
- **脉冲执行**：`CMLP`、`DCMLP`（仅在条件上升沿执行一次）

示例：

plaintext

```
LD    X0
CMLP  K5   D0      ; 仅在X0按下瞬间，将5取反后传送到D0
```



------

#### 4. 典型应用示例

##### 例1：控制输出取反

plaintext

```
MOV   K5   K2Y0    ; 之前例子：Y0=1,Y2=1，其余0
CML   K5   K2Y0    ; 将5(0101)取反得1010 → Y0=0,Y1=1,Y2=0,Y3=1
```



| Y点   | 位值 | 状态                  |
| :---- | :--- | :-------------------- |
| Y0    | 0    | OFF                   |
| Y1    | 1    | ON                    |
| Y2    | 0    | OFF                   |
| Y3    | 1    | ON                    |
| Y4~Y7 | 1    | ON（因为高4位也取反） |

##### 例2：产生逻辑非的掩码

假设 D0 = `0x0F0F` (0000111100001111)，执行 `CML D0 D10` 后 D10 = `0xF0F0` (1111000011110000)。常用于将某段数据取反后参与运算。

##### 例3：与 MOV 配合实现部分位取反

plaintext

```
MOV   D0   D1       ; 备份原值
CML   D0   D2       ; 取反后的值
```



------

#### 5. CML 与 MOV 的区别

| 指令 | 操作       | 示例（S=5, 16位） | 结果 D                      |
| :--- | :--------- | :---------------- | :-------------------------- |
| MOV  | 直接传送   | `MOV K5 D0`       | D0=5 (0x0005)               |
| CML  | 取反后传送 | `CML K5 D0`       | D0 = 0xFFFA（16位取反，-6） |

注意：CML 是对**所有位**（包括高位）取反。5 的16位二进制是 `0000 0000 0000 0101`，取反后为 `1111 1111 1111 1010`，即十进制 -6。

------

#### 6. 使用注意事项

1. **取反的范围**
   CML 对源数据的**全部有效位**（16位或32位）进行取反，不是仅低几位。如果只希望取反低4位，可以先使用 AND 等指令掩码，再用 CML，或者使用位组合元件。

2. **位组合元件的取反**

   plaintext

   ```
   CML   K1X0   K1Y0   ; 将 X0~X3 取反后输出到 Y0~Y3
   ```

   

   如果 X0=1,X1=0,X2=1,X3=0，则 Y0=0,Y1=1,Y2=0,Y3=1。

3. **负数处理**
   取反后数值可能变为负数（最高位为1）。如果后续用作计数器设定值或比较值，需注意符号。

4. **脉冲执行**
   若不需要每个周期都取反传送，建议使用 `CMLP` 或 `DCMLP`，避免无意义的重复操作。

5. **与 NOT 指令的区别**

   - `NOT` 指令用于逻辑运算，结果存于累加器（FX3U中不常用）。
   - `CML` 是传送指令，将取反后的数据直接写入目标寄存器。

------

#### 7. 常见应用场景

- **生成互补信号**：例如步进电机的方向控制（正转/反转信号互为取反）。
- **数据加密/校验**：简单的按位取反加密。
- **PLC与外部设备接口**：某些设备要求数据按位取反后发送。
- **避免使用专用取反线圈**：当需要对一组输出同时取反时，一条 CML 指令比多个 `OUT` 或 `ALT` 更简洁。

------

#### 8. 总结

| 指令 | 全称                   | 功能               | 常用搭配       |
| :--- | :--------------------- | :----------------- | :------------- |
| CML  | Complement Move        | 按位取反后传送     | 脉冲执行 CMLP  |
| DCML | Double Complement Move | 32位按位取反后传送 | 脉冲执行 DCMLP |

**一句话记忆**：`CML` 相当于先取反再 `MOV`，适合需要反向数据输出的场合。使用时注意位宽和符号位的影响。



# **BMOV（成批传送）指令**-Block Move



用于将一段连续的数据寄存器（或位元件）的内容批量复制到另一段连续的软元件中。

------

#### 1. 指令格式

**BMOV** 的基本格式：

plaintext

```
BMOV   S   D   n
```



- **S**：源起始元件（数据寄存器 D、位组合 KnX/KnY/KnM、定时器 T、计数器 C 的当前值）
- **D**：目标起始元件（数据寄存器 D、位组合 KnY/KnM 等）
- **n**：传送个数（常数 K/H，或数据寄存器 D 间接指定）

##### 示例

plaintext

```
BMOV   D0   D10   K5    ; 将 D0~D4 的内容传送到 D10~D14
BMOV   K1X0 K1Y0  K2    ; 将 X0~X3 和 X4~X7 分别传送到 Y0~Y3、Y4~Y7（两个4位组）
```



------

#### 2. 功能说明

BMOV 将 **以 S 为起始的连续 n 个元件** 的内容 **原样复制** 到 **以 D 为起始的连续 n 个元件** 中。传送过程中不改变源数据，目标区域原有数据被覆盖。

- **16位传送**：每个元件为16位（如 D、T、C、KnX 等）。
- **32位传送**：没有专门的 32位成批传送指令（需要32位时，使用 `DMOV` 结合循环，或直接用 `BMOV` 但每个元件仍为16位）。
- **脉冲执行**：`BMOVP`（仅在条件上升沿执行一次）。

------

#### 3. 适用元件类型

| 元件类型             | 可作为 S | 可作为 D | 说明                                                         |
| :------------------- | :------- | :------- | :----------------------------------------------------------- |
| 数据寄存器 D         | ✅        | ✅        | 最常用                                                       |
| 定时器 T 当前值      | ✅        | ✅        | 传送时需注意 T 的当前值范围                                  |
| 计数器 C 当前值      | ✅        | ✅        | 16位/32位均可（但 BMOV 按16位处理）                          |
| 位组合 K1~K4（16位） | ✅        | ✅        | 如 K1X0、K2Y0、K4M0                                          |
| 位组合 K5~K8（32位） | ⚠️        | ⚠️        | BMOV 处理16位，32位需用 `DMOV` 分次或使用 `DBMOV`（FX3U支持？需查手册） |
| 文件寄存器 R         | ✅        | ✅        | FX3U支持                                                     |

**注意**：BMOV 不能直接用于传送位软元件（X、Y、M）的**单个点**，但可通过 `Knm` 形式成批传送位状态。

------

#### 4. 典型示例

##### 例1：数据寄存器批量备份

plaintext

```
BMOV   D100   D200   K20    ; 将 D100~D119 备份到 D200~D219
```



##### 例2：位组合传送（输入映象到输出）

plaintext

```
BMOV   K2X0   K2Y0   K1     ; 将 X0~X7 的状态传送到 Y0~Y7
```



这里 `K2X0` 表示 8 个输入点，`K2Y0` 表示 8 个输出点，`n=1` 表示传送 1 个“单位”，但这里的单位是 **16位**？
**重要**：对于位组合，`n` 表示的是 **要传送的16位字的个数**，而不是位的个数。

- `K2X0` 已经占用 8 位，但它仍然作为一个16位字处理（高8位为0）。
- 若写 `BMOV K1X0 K1Y0 K2`，则传送 2 个 16位字：第一个字传送 X0~X3 到 Y0~Y3，第二个字传送 X4~X7 到 Y4~Y7。

**建议**：对于位组合的成批传送，通常使用 `BMOV` 时 `n=1` 即可，因为 `Kn` 已指定了位数。

##### 例3：清空一段寄存器

plaintext

```
MOV    K0   D0        ; 先清零一个寄存器
BMOV   D0   D1   K9   ; 将 D0 的0传送到 D1~D9（共10个寄存器全0）
```



或者直接使用 `FMOV`（多点传送）更合适。

------

#### 5. BMOV 与 MOV 的区别

| 指令 | 传送数量 | 源/目标形式      | 典型场景             |
| :--- | :------- | :--------------- | :------------------- |
| MOV  | 1 个数据 | 单个元件或位组合 | 单一数值复制         |
| BMOV | n 个数据 | 连续的元件块     | 数据块复制、数组操作 |

**关系**：`BMOV S D 1` 等同于 `MOV S D`（但 MOV 支持更多元件类型）。

------

#### 6. 使用注意事项

1. **源与目标区域重叠时的处理**
   如果源区域和目标区域有重叠，BMOV 会按照 **从源起始地址向高地址方向依次传送** 的方式进行，可能导致尚未传送的数据被覆盖。

   - 当 **S < D**（源地址低于目标地址）且重叠时，应从高地址向低地址传送？实际FX中 BMOV 的处理规则：
     **若 S < D 且 (S + n - 1) ≥ D，则传送方向自动改为从高地址向低地址**，以避免覆盖。
   - 若 S > D 且重叠，则从低地址向高地址传送。
   - 总之，BMOV 内部会自动处理重叠情况，无需用户干预。但为了可读性，尽量避免重叠。

2. **位组合传送时的 `n` 含义**
   `n` 是 **16位字** 的个数。例如：

   plaintext

   ```
   BMOV   K2M0   K2M10   K3    ; 传送 3×16=48 位，即从 M0~M47 到 M10~M57
   ```

   

   确保软元件范围足够，且不要跨区（如 M 只有 768~1024 个）。

3. **32位数据的成批传送**
   BMOV 按16位处理。如果需要传送 32 位数据块（如 D0/D1 组成 32 位，要复制到 D10/D11），可以：

   - 使用 `DMOV` 配合循环（如 FOR-NEXT）。
   - 使用 `DBMOV`（FX3U 支持？查阅手册：FX3U 有 `DMBOV`？实际 FX3U 无 DBMOV，但可使用 `BMOV` 并将 n 乘以 2，但注意高低字顺序不变。）
   - 推荐：用 `RCL` 或指针变址，或者使用 `MOV` 两个两个地传。

4. **脉冲执行**
   如果不需要每个扫描周期都执行成批传送（例如只在启动时备份），务必使用 `BMOVP`，避免大量数据传送影响扫描周期。

5. **传送个数 n 的范围**

   - n ≤ 4096（FX3U 数据寄存器 D 范围）。
   - 若 n 用数据寄存器间接指定（如 `BMOV D0 D10 D100`），需确保 D100 值有效。

------

#### 7. 与 FMOV（多点传送）的区别

- **BMOV**：从源区域复制数据到目标区域（源和目标可以是不同区域，也可以是同一区域但不同位置）。
- **FMOV**：将一个源数据**填充**到目标区域的连续 n 个元件中（所有目标得到相同的值）。

plaintext

```
FMOV   K0   D10   K5   ; 将 D10~D14 全部设为 0
BMOV   D0   D10   K5   ; 将 D0~D4 的值分别复制到 D10~D14（可能各不相同）
```



------

#### 8. 典型应用场景

- **配方数据加载**：将配方表（如 D1000~D1099）一次性传送到工作区（D0~D99）。
- **历史数据记录**：将当前数据批量保存到历史缓冲区。
- **显示刷新**：将一组数据寄存器的值批量传送到显示缓冲区（如 BCD 码输出）。
- **输入/输出映象批量更新**：用 BMOV 将输入区的位组合复制到输出区（但需注意扫描时序）。

------

#### 9. 总结

| 指令 | 全称       | 功能                          | 常用搭配       |
| :--- | :--------- | :---------------------------- | :------------- |
| BMOV | Block Move | 成批传送连续 n 个数据         | 脉冲执行 BMOVP |
| FMOV | Fill Move  | 将一个数据填充到连续 n 个元件 | 初始化数据区   |

**一句话记忆**：`BMOV` 是数据块的批量复制，适合数组、表格、缓冲区操作；注意 `n` 是元件个数（16位单位），且会自动处理重叠区域。配合脉冲执行可大幅提高程序效率。



# **FMOV（多点传送）指令-Fill Move**





也称为填充指令，用于将一个源数据复制到目标区域的连续多个元件中。

------

#### 1. 指令格式

**FMOV** 的基本格式：

plaintext

```
FMOV   S   D   n
```



- **S**：源数据（常数 K/H、数据寄存器 D、定时器 T/计数器 C 当前值、位组合等）
- **D**：目标起始元件（数据寄存器 D、位组合 KnY/KnM 等）
- **n**：传送个数（常数 K/H，或数据寄存器 D 间接指定）

### 示例

plaintext

```
FMOV   K0    D0    K10   ; 将 D0~D9 全部清零
FMOV   D100  D200  K5    ; 将 D100 的值复制到 D200~D204（所有目标值相同）
FMOV   K1    K4Y0  K2    ; 将 1 填充到 Y0~Y15（每个4位组都得到1）
```



------

#### 2. 功能说明

FMOV 将 **源数据 S** 复制到 **以 D 为起始的连续 n 个元件** 中，使这些目标元件全部获得相同的值。源数据本身不变，目标区域原有数据被覆盖。

- **16位传送**：每个元件为16位（如 D、T、C 当前值、Kn 位组合等）。
- **32位传送**：**DFMOV** 用于32位数据填充（占用两个连续寄存器）。
- **脉冲执行**：`FMOVP`（仅在条件上升沿执行一次）。

------

#### 3. 适用元件类型

| 元件类型          | 可作为 S | 可作为 D | 说明                           |
| :---------------- | :------- | :------- | :----------------------------- |
| 常数（K/H）       | ✅        | ❌        | 最常用作清零或预设值           |
| 数据寄存器 D      | ✅        | ✅        | 批量赋值                       |
| 定时器 T 当前值   | ✅        | ✅        | 批量设置定时器当前值（不常用） |
| 计数器 C 当前值   | ✅        | ✅        | 批量设置计数器当前值           |
| 位组合（KnY/KnM） | ✅        | ✅        | 批量控制输出或中间继电器       |
| 文件寄存器 R      | ✅        | ✅        | FX3U支持                       |

**注意**：D 不能是只读元件（如输入 X、特殊寄存器 D8000 部分只读）。

------

#### 4. 典型示例

##### 例1：数据寄存器区域清零（最常用）

plaintext

```
FMOV   K0    D0    K100   ; 将 D0~D99 全部设为 0
```



##### 例2：用同一个预设值初始化多个寄存器

plaintext

```
MOV    K1234  D100        ; 预设值存入 D100
FMOV   D100   D200  K20   ; 将 D200~D219 都设为 1234
```



##### 例3：批量控制输出点（将一组Y全部置ON或OFF）

plaintext

```
FMOV   K1    K4Y0  K1     ; 将 Y0~Y15 全部置为 ON（因为 K1=0001，填充到16位时每个位都重复？不对！）
```



**重要澄清**：对于位组合目标 `KnY`，FMOV 填充的是 **16位字的数值**，而不是按位填充。

- `K4Y0` 占用 Y0~Y17（16个点），是一个16位字。
- `FMOV K1 K4Y0 K1` 会将 K1（0x0001）写入这个字，结果只有 Y0=ON，Y1~Y15=OFF。
- 若想将所有 16 个 Y 都置为 ON，应使用 `FMOV K65535 K4Y0 K1`（16位全1）。
- 若想将 8 个 Y（K2Y0）全部置为 ON，可用 `FMOV K255 K2Y0 K1`（8位全1）。

**批量置ON的正确方法**：

plaintext

```
FMOV   HFFFF  K4Y0  K1   ; Y0~Y15 全 ON
FMOV   HFF    K2Y0  K1   ; Y0~Y7 全 ON
```



##### 例4：32位填充（DFMOV）

plaintext

```
DFMOV  K100000  D0   K5   ; 将 (D1,D0)=100000 填充到 (D3,D2)、(D5,D4)...(D9,D8) 共5个32位数据块
```



注意：DFMOV 中 D 占用连续 2n 个数据寄存器。

------

#### 5. FMOV 与 BMOV 的区别

| 指令 | 源数据形式 | 目标区域内容                               | 典型场景             |
| :--- | :--------- | :----------------------------------------- | :------------------- |
| FMOV | 单个数据   | 所有目标元件都等于该数据                   | 清零、批量设同一个值 |
| BMOV | 连续数据块 | 目标区域是源区域的副本（每个位置可能不同） | 数据块复制、备份     |

**关系**：`FMOV S D n` 相当于 `for i=0 to n-1: D[i] = S`。

------

#### 6. 使用注意事项

1. **目标元件数量 n 的范围**
   - n ≤ 4096（FX3U 数据寄存器范围）。
   - 若 n 用数据寄存器指定（如 `FMOV D0 D10 D100`），需确保 D100 值在有效范围内，否则可能超出软元件边界导致错误。
2. **位组合目标时的数值范围**
   - 目标为 `K1Y0`（4位）时，S 的有效范围是 0~15（0xF）。
   - 目标为 `K2Y0`（8位）时，S 的范围 0~255。
   - 目标为 `K4Y0`（16位）时，S 的范围 0~65535。
     超出范围的高位会被截断。
3. **32位填充**
   - 使用 `DFMOV`，注意目标地址占用双倍数量。例如 `DFMOV K0 D0 K10` 会占用 D0~D19 共20个寄存器（10个32位数）。
   - 如果后续程序使用了 D1、D3 等，需注意不要冲突。
4. **脉冲执行**
   建议使用 `FMOVP` 避免每个扫描周期都重复填充（尤其当 n 较大时，会显著增加扫描时间）。
5. **与 RST/ZRST 的区别**
   - `RST D0` 只能复位一个数据寄存器。
   - `ZRST D0 D99` 可以批量复位（清零）一段数据寄存器，但 ZRST 是专门用于位元件或数据寄存器的复位指令，功能与 `FMOV K0 D0 K100` 类似，但 ZRST 执行速度可能更快（指令优化）。通常清零时 ZRST 和 FMOV 均可，但 ZRST 更直观。
   - 但 FMOV 可以填充非零值，更灵活。
6. **源数据为寄存器时的注意事项**
   如果 S 是数据寄存器，且在执行 FMOV 过程中该寄存器的值发生变化（如被中断程序修改），可能导致部分目标得到旧值、部分得到新值。建议在填充前先将 S 存入临时寄存器，或使用 DI 关闭中断。

------

#### 7. 常见应用场景

- **批量清零**：`FMOV K0 D0 K100` 或 `ZRST D0 D99`。
- **批量预设**：将同一个参数（如报警阈值、定时器预设值）赋给多个控制回路。
- **初始化输出**：将一组输出点设为确定状态（如全部 OFF）。
- **波形/数据生成**：用 FMOV 快速填充一块内存为恒定值，然后逐点修改。

------

#### 8. 总结

| 指令  | 功能                        | 数据长度 | 常用搭配        |
| :---- | :-------------------------- | :------- | :-------------- |
| FMOV  | 将源数据填充到连续 n 个目标 | 16位     | 脉冲执行 FMOVP  |
| DFMOV | 32位数据填充                | 32位     | 脉冲执行 DFMOVP |

**一句话记忆**：`FMOV` 是“批量赋值”，让一片连续元件都等于同一个值；`BMOV` 是“批量复制”，把源数据块完整拷贝到目标区。需要清零或统一设置时用 FMOV，需要搬运不同数据块时用 BMOV。
